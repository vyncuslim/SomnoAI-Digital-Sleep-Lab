隐私权政策
Privacy Policy

版本号：v2024.03.R3 | 最后修订：2024年3月22日

1. 我们如何看待您的隐私

SomnoAI Digital Sleep Lab（以下简称“本应用”或“我们”）深知健康数据的敏感性。本应用被设计为一个“端侧优先” (Client-Side First) 的实验平台。我们的核心价值观是：您的数据属于您，且仅属于您的设备。我们不会在任何云端服务器上持久化存储您的原始生理信号。

2. 数据收集深度说明

当且仅当您明确点击“接入 Google Fit”并完成 OAuth 授权后，我们才会请求访问以下作用域 (Scopes) 下的数据：

睡眠架构数据

访问 fitness.sleep.read 以提取睡眠分段。这包括您入睡、醒来以及处于深睡、REM、浅睡阶段的精确时间戳，用于生成睡眠架构图谱。

生物节律与代谢数据

访问 fitness.heart_rate.read 和 fitness.activity.read。我们提取静息心率与周期能耗数值，以分析代谢效率对睡眠恢复的影响。

3. Google API 有限使用政策 (Limited Use)

本应用对从 Google API 接收到的信息的使用和传输将严格遵守 Google API 服务用户数据政策，包括其“有限使用”要求：

  • 不用于广告：绝不将您的 Google Fit 数据共享或出售给广告平台、数据经纪人或任何第三方营销机构。
  • 不用于个性化推荐：数据仅用于本应用内显示的睡眠洞察，不会用于构建您的跨平台广告画像。
  • 不用于信贷/保险：您的健康指标不会被用于评估您的信贷资格、保险定价或任何具有法律效力的决定。

4. 数据处理逻辑与 AI 引擎

本应用集成了 Google Gemini 系列模型（如 gemini-3-flash-preview）来生成个性化的睡眠洞察：

  • 匿名化处理：在发送数据给 AI 引擎前，我们会去除所有身份识别符（如姓名、邮箱、地理位置）。
  • 仅处理指标：AI 仅接收数值化的睡眠分数、心率均值和能量消耗百分比，而非原始的生物波形数据。
  • 非训练声明：我们通过 API 配置确保您的数据不会被用于训练 Google 的基础大型语言模型。

5. 数据存储、安全与生命周期

临时性存储：为了提升交互流畅度，我们会将提取的数据临时缓存于浏览器的 sessionStorage 中。这种存储方式具有临时性，当您关闭浏览器标签页或手动点击“退出”时，缓存将被操作系统自动抹除。

传输安全：所有从 Google 服务器到您浏览器的通信均采用 TLS 1.2/1.3 加密协议。在数据流经本应用的前端逻辑时，始终处于隔离的沙箱环境内。

6. 用户的绝对控制权

您可以随时行使以下权利：

  • 访问与导出：您可以在应用界面实时查看所有提取的数据项。
  • 撤销授权：随时在您的 Google 账号安全设置 中撤销对 SomnoAI 的授权，撤销后应用将立即失去访问权限。
  • 一键抹除：点击应用内“设置”->“退出并重载”，将立即清除本地所有 Session 缓存。

7. 儿童隐私保护

本应用主要面向追求健康生活的成年用户（18岁及以上）。我们不会故意收集 13 岁以下儿童（或您所在司法管辖区规定的相关年龄）的个人数据。如果您发现有未成年人未经授权使用本应用，请通过下方邮件联系我们。
