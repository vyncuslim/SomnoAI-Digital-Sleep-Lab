隐私权政策

最后更新日期： 2024年5月22日

本政策旨在向您详尽说明 SomnoAI 数字化睡眠实验室 (sleepsomno.com) 如何访问、使用、存储及处理通过 Google API 授权获取的健康数据。我们深知生理数据的敏感性，始终坚持隐私优先、公开透明的原则。

Google API 有限使用披露

本应用对从 Google API 接收的信息的使用和传输将严格遵守 Google API 服务用户数据政策，包括其中的“有限使用”要求。我们承诺不将接收到的健康数据用于广告投放、个人画像构建、信用评估或任何形式的第三方数据交易。

核心披露章节 一

1. 数据访问说明 (Data Accessed)

当您授权连接 Google Fit 后，本应用将基于实现核心分析功能的“最小必要原则”访问以下数据范围：

  • 睡眠片段数据 (fitness.sleep.read)： 我们访问您的睡眠分期（如深睡、REM、浅睡、清醒）及各阶段的精确持续时间。
  • 心率生理指标 (fitness.heart_rate.read)： 我们访问您在静息及睡眠期间的心率数据（BPM），用于评估自主神经系统的恢复状况。
  • 能量消耗数据 (fitness.activity.read)： 我们访问您的每日活动卡路里消耗，用于多维度的生理负荷关联分析。
  • 基本个人资料： 仅读取您的姓名和头像，用于在本地实验室内为您提供个性化的交互体验。
核心披露章节 二

2. 数据使用方式 (Data Usage)

我们访问上述 Google 用户数据并非为了收集，而是为了进行即时的数字化实验处理，具体用途如下：

  • 本地化指标计算： 利用内置算法，在您的浏览器前端直接将心率和睡眠数据转化为“睡眠评分”与“身体恢复指数”。
  • AI 驱动的深度洞察： 经过脱敏处理的生理数值（不含您的姓名、邮箱等身份识别信息）将被发送至 Google Gemini 模型，用于合成个性化的睡眠优化建议和次日的认知负荷预测。
  • 健康趋势可视化： 将历史数据聚合成动态图谱，辅助您进行长期的健康趋势自我监测。
核心披露章节 三

3. 数据存储、保留与安全 (Storage & Retention)

本应用采用创新的 “零后端 (Zero-Backend)” 架构，确保数据主权始终归用户所有：

  • 无云端存储： 您的任何 Google 用户数据 绝不会 被上传、传输或存储到我们的任何服务器上。
  • 本地会话缓存： 数据仅暂时驻留在您当前浏览器设备的 sessionStorage（临时会话存储）中。
  • 即时销毁机制： 一旦您关闭网页浏览器标签页、刷新页面或点击“登出”，所有临时存储的生理指标将立即被系统彻底抹除，无法恢复。
核心披露章节 四

4. 数据共享声明 (Sharing)

我们绝不共享您的数据。 本应用不会向任何第三方（包括广告代理商、数据经纪人或非关联研究机构）转售、披露或共享您的原始或经过处理的 Google 健康数据。所有数据流转仅限于本地计算及必要的脱敏 AI 分析接口调用。

用户控制权利

5. 权限撤回与管理

您可以随时在应用设置中点击“退出并重载”以清除所有本地缓存。此外，您随时可以通过您的 Google 账号权限管理页面 永久撤销对 SomnoAI 的授权。
