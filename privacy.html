Somno Digital Sleep Lab 运行在 零后端边缘加密架构 之上。我们深知生理数据的敏感性，本协议旨在透明化说明我们如何通过 Google Fit API 访问数据以及我们对 Google “有限使用”政策的坚定承诺。

Google API 有限使用披露

Somno 对从 Google API 接收的信息的使用和向任何其他应用的转移，将遵守 Google API 服务用户数据政策，包括其中的“有限使用”要求。

核心承诺： 我们绝不将您的健康数据用于广告投放、用户画像或共享给任何第三方数据经纪人。所有数据仅用于在本应用内提供睡眠分析。

1. 遥测架构 (数据访问范围)

为了合成您的“睡眠架构图谱”，实验室请求访问以下只读数据集：

  • DATA_SEG_01
    睡眠阶段 (com.google.sleep.segment)

    读取深度睡眠、快速眼动 (REM) 和浅睡阶段。这是 AI 解析睡眠架构的基础输入。

  • DATA_SEG_02
    心率遥测 (com.google.heart_rate.bpm)

    读取静息心率 (RHR) 及波动，用于评估生理恢复程度和心血管系统在睡眠中的状态。

  • DATA_SEG_03
    代谢流量 (com.google.calories.expended)

    将能量消耗与睡眠效率关联，从代谢维度提供多维度的健康洞察。

2. 处理逻辑 (数据用途)

访问的数据通过以下流程进行处理：

  • 睡眠架构分析： 计算睡眠阶段比例，生成主观“睡眠分数”。
  • 生理投影： 利用本地 AI 逻辑预测次日的认知状态和体力恢复情况。
  • 神经合成： 将脱敏后的指标作为加密瞬时负载发送至 Gemini API，生成个性化的健康建议。

3. 零后端与持久化策略

本实验室采用 隐私优先架构，确保数据仅在您的浏览器中流动：

传输安全

数据通过 SSL/TLS 直接从 Google 服务器同步到您的浏览器。Somno 不设立任何后端数据库。

瞬时存储

生理指标仅暂存于内存 (sessionStorage) 中，页面关闭或手动退出时即刻物理销毁。
